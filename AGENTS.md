# Agent 长期记忆规则

## 项目特定规则

### 代码风格
1. 所有C#代码使用PascalCase命名类和方法，camelCase命名变量
2. 使用4个空格进行缩进，不要使用制表符
3. 遵循《杀戮尖塔2》mod的现有代码风格
4. 避免在代码中添加注释，除非必要

### 文档规范
1. 所有说明性文档中避免使用emoji表情符号
2. 使用文字描述和标准符号
3. 保持文档的专业性和可读性

### 开发流程
1. 实现新功能前先更新TODO.md
2. 完成功能后更新README.md中的进度
3. 优先实现高优先级功能
4. 每个功能完成后运行相关测试
5. 每次完成重要修改后，在git中提交改动
6. 及时使用`git push origin <branch>`提交到远程仓库

### 环境配置
1. 项目在Windows环境下开发，使用PowerShell而不是bash
2. 注意PowerShell命令语法与bash的区别
3. 使用`dir`代替`ls`，使用PowerShell兼容的命令

## 构建和测试命令

### 构建命令
```bash
dotnet build
```

### 测试命令
```bash
# 需要先确认项目的测试框架
# 如果是NUnit：
dotnet test
# 如果是xUnit：
dotnet test
```

### 代码质量检查
```bash
# 代码格式化（如果配置了）
dotnet format
# 静态分析（如果配置了）
dotnet analyze
```

## 项目特定偏好

### 文件组织
1. 将相关文件放在同一目录下
2. 遵循现有的目录结构
3. 新文件添加到适当的目录

### 依赖管理
1. 优先使用BaseLib提供的功能
2. 避免引入不必要的依赖
3. 保持与《杀戮尖塔2》版本的兼容性

### 错误处理
1. 使用try-catch处理可能失败的操作
2. 记录错误到日志文件
3. 提供用户友好的错误信息

## 记忆更新规则
1. 当发现新的构建或测试命令时，更新此文件
2. 当项目规范变化时，更新此文件
3. 定期检查此文件以确保规则仍然适用
