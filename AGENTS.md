# Agent 长期记忆规则

## 项目特定规则

### 代码风格
1. 所有C#代码使用PascalCase命名类和方法，camelCase命名变量
2. 使用4个空格进行缩进，不要使用制表符
3. 遵循《杀戮尖塔2》mod的现有代码风格
4. 为每个函数和类添加必要的注释说明

### 文档规范
1. 所有说明性文档中避免使用emoji表情符号
2. 保持文档的专业性和可读性

### 开发要求

1. 任何不明确的功能，或者不了解的内容，请总是先尝试搜索网络，然后向用户提问。
2. 遇到代码中的问题时，请参考[这个网站](https://glitchedreme.github.io/SlayTheSpire2ModdingTutorials/docs/02-install-view-source-and-patch/)中的内容尝试解决。

### 开发流程
1. 实现新功能前先更新TODO.md
2. 完成功能后更新README.md中的进度
3. 每个功能完成后运行相关测试
4. 每次完成重要修改后，先询问用户的审查意见，再使用git提交改动

### 环境配置
1. 项目在Windows环境下开发，使用PowerShell而不是bash。注意PowerShell命令语法与bash的区别

## 项目特定偏好

### 文件组织
1. 将相关文件放在同一目录下
2. 遵循现有的目录结构
3. 新文件添加到适当的目录

### 依赖管理
1. 优先使用BaseLib提供的功能
2. 保持与《杀戮尖塔2》版本的兼容性

### 错误处理
1. 使用try-catch处理可能失败的操作
2. 记录错误到日志文件
3. 提供用户友好的错误信息

## 项目目录结构说明

### 工具目录 (`E:\slay_the_spire_mod\tools\`)
1. **反编译工具存储** - 包含游戏反编译所需的工具
2. **dnSpy工具** - 位于 `tools\dnspy\` 目录，用于查看和反编译.NET程序集
3. **ilspycmd工具** - 已安装为dotnet全局工具，用于命令行反编译
4. **工具管理** - 所有开发相关工具集中存放于此目录

### 反编译结果目录 (`E:\slay_the_spire_mod\decompiled\`)
1. **游戏代码参考** - 包含《杀戮尖塔2》游戏主程序的反编译C#代码
2. **代码结构** - `sts2_split/` 目录包含3305个按命名空间组织的C#文件
3. **开发指南** - 包含README.md和DEVELOPMENT_GUIDE.md文档，指导如何参考游戏代码
4. **关键系统代码** - 包含卡牌、战斗、实体、模型等核心系统的实现参考

### 使用说明
1. **代码参考** - 开发模组时可参考 `decompiled\sts2_split\` 中的游戏实现
2. **工具使用** - 需要反编译其他dll时使用 `tools\` 目录中的工具
3. **版本控制** - 这些目录不应提交到git仓库，仅用于本地开发参考
