# WorldlineViewer 网页可视化工具 - 开发计划

## 项目概述
为WorldlineTracker项目开发一个网页可视化工具，用于展示战斗过程记录。用户可以通过上传JSON文件，在网页上以节点+连线的方式可视化战斗事件。

## 技术栈选择
- **前端框架**: Vue.js 3.x
- **可视化**: 纯Canvas实现（无外部库依赖）
- **文件处理**: 纯前端JavaScript（无后端服务器）
- **样式**: CSS3 + Flexbox布局

## 核心功能需求

### 1. 文件上传功能
- 支持拖拽上传和点击选择文件
- 仅接受JSON格式文件
- 实时验证文件格式和内容
- 显示上传状态和错误提示

### 2. 数据可视化
- 每个战斗事件显示为圆形节点
- 节点摘要格式：`Tx: 动作类型中文`
  - 示例：`T1: 出牌`、`T2: 敌人动作`、`T3: 使用药水`
- 节点按时间顺序线性连接
- 不同动作类型使用不同颜色区分
- 支持节点自由拖拽

### 3. 交互功能
- 点击节点显示完整详细信息在左下角文本框
- 节点拖拽后位置保存到浏览器本地存储
- 支持画布缩放和平移（可选）
- 响应式布局适应不同屏幕尺寸

### 4. 数据显示
- 左下角文本框显示选中节点的完整JSON数据
- 格式化显示，包含所有字段信息
- 支持复制详细信息功能

## 技术设计细节

### 数据模型设计
```javascript
// 节点数据模型
class Node {
  id: string
  turnNumber: number      // 回合数，格式：T1, T2, T3
  actionType: string      // 动作类型中文
  originalData: object    // 原始JSON数据
  x: number              // 画布X坐标
  y: number              // 画布Y坐标
  radius: number         // 节点半径
  color: string          // 节点颜色（基于动作类型）
}

// 连线数据模型
class Connection {
  sourceNodeId: string
  targetNodeId: string
}

// 可视化数据模型
class VisualizationData {
  nodes: Node[]
  connections: Connection[]
  battleInfo: object      // 战斗基本信息
}
```

### 颜色映射方案
根据动作类型分配不同颜色：
- `出牌`: #4CAF50 (绿色)
- `使用药水`: #2196F3 (蓝色)
- `遗物触发`: #FF9800 (橙色)
- `敌人动作`: #F44336 (红色)
- `回合开始`: #9C27B0 (紫色)
- `回合结束`: #607D8B (灰色)
- `造成伤害`: #FF5722 (深橙色)
- `其他动作`: #795548 (棕色)

### 布局算法
1. **初始布局**: 按时间顺序水平排列
2. **间距计算**: 根据节点数量和画布大小动态计算
3. **拖拽支持**: 保存拖拽后位置到localStorage
4. **重新布局**: 提供"重置布局"功能

## 技术挑战与解决方案

### 挑战1：Canvas性能优化
**问题**: 大量节点和连线可能导致性能问题
**解决方案**:
- 使用requestAnimationFrame进行动画
- 实现脏矩形更新，只重绘变化区域
- 添加节点数量限制和警告
- 实现虚拟滚动（如果节点过多）

### 挑战2：跨浏览器兼容性
**问题**: 不同浏览器对Canvas和文件API支持不同
**解决方案**:
- 使用现代浏览器API的polyfill
- 添加功能检测和降级方案
- 测试主流浏览器（Chrome, Firefox, Edge）

### 挑战3：数据格式兼容性
**问题**: WorldlineTracker JSON格式可能变化
**解决方案**:
- 实现灵活的数据解析器
- 添加版本检测和兼容处理
- 提供详细的错误信息
- 支持示例数据文件

### 挑战4：用户体验优化
**问题**: 复杂的可视化可能让用户困惑
**解决方案**:
- 添加新手引导和提示
- 实现逐步加载动画
- 提供布局重置功能
- 添加键盘快捷键支持
